# PA2: Code-Switched Speech Transcription and Telugu Voice Cloning

**Speech Understanding - Programming Assignment 2**

---

## Overview

This repository contains the complete implementation of a pipeline that:
1. Transcribes code-switched Hindi-English (Hinglish) lecture audio
2. Re-synthesizes the content in Telugu (Low-Resource Language) using zero-shot voice cloning

All code is written using Python and PyTorch only.

## Part-by-Part Summary

### Part I: Robust Code-Switched Transcription

- **Cell 1** - Preprocessing: stereo to mono, resample to 16kHz, gain normalization, trim to 10 min
- **Cell 2** - Denoising (Task 1.3): PyTorch STFT-based spectral subtraction, alpha=3.5, beta=0.005
- **Cell 3** - Multi-Head LID (Task 1.1): custom LSTM-free MFCC classifier with English + Hindi heads, Macro F1 = 1.0
- **Cell 4** - Constrained Decoding (Task 1.2): Whisper medium with N-gram logit biasing (gamma=3.0) and custom LogitFilter hook

### Part II: Phonetic Mapping and Translation

- **Cell 5** - IPA (Task 2.1): Manual Hinglish G2P layer with Unicode-based language routing per token
- **Cell 5** - Telugu Translation (Task 2.2): 500-word parallel technical dictionary, Telugu chosen as LRL

### Part III: Zero-Shot Voice Cloning

- **Cell 6** - D-Vector (Task 3.1): 3-layer LSTM + mean pooling + L2 norm, 256-dim speaker embedding
- **Cell 6** - DTW Prosody Warping (Task 3.2): PYIN F0 + RMS energy, O(NM) DTW alignment, student-to-lecture style transfer
- **Cell 6** - TTS Synthesis (Task 3.3): Telugu gTTS with DTW-warped prosody post-processing at 22050 Hz

### Part IV: Adversarial Robustness

- **Cell 8** - Anti-Spoofing CM (Task 4.1): 80-dim LFCC features, binary classifier, EER evaluation
- **Cell 8** - FGSM Attack (Task 4.2): Feature-space FGSM, epsilon sweep, SNR > 40 dB constraint

## Key Design Choices

1. **LID training data**: gTTS Hindi used due to lack of labelled Hinglish corpus for this specific lecture
2. **Logit bias hook**: Undocumented `logit_filter` parameter in Whisper's `DecodingOptions` used for constrained decoding
3. **IPA routing**: Unicode block detection (Devanagari U+0900-U+097F) for per-token language routing
4. **DTW truncation**: 1000-frame (62.5s) window used for computational feasibility; sufficient for quasi-periodic lecture prosody
5. **LFCC over MFCC**: Linear filterbank preserves high-frequency resolution for TTS artifact detection

## Low-Resource Language

**Telugu** was selected as the target LRL.
- Dravidian language family
- ~80 million speakers
- Fully phonemic script
- Limited ASR/TTS resources compared to Hindi/English
- Fallback: Santhali (if Telugu TTS fails)
